# Project Timeline  
**Mapping Competence Needs in Sweden’s Labor Market for HVE Institutions**  
*Timeline is subject to change.*

---

## PHASE 1 — Project Initiation  
**Goal:** Align on purpose, scope, and expected outcomes.  
**Mindset:** Define the “data product” you’re building.

**Actions:**  
- Read and summarize the project’s purpose.  
- Identify end-users: HVE institutions, companies, recruiters, professionals.  
- Define the final output (report + charts + insights).  
- Agree on internal roles (PM, analyst, outreach, etc.).  
- Set up shared folder structure (OneDrive, Discord, GitHub).  
- Create a communication plan for outreach and follow-ups.

**Deliverable:**  
A one-page scope summary in your own words.

---

## PHASE 2 — Planning Phase  
**Goal:** Build the backbone of the study.  
**Mindset:** Design your ETL architecture.

**Actions:**  
- Break the project into streams (Survey Data, Public Data, Interviews).  
- Design Microsoft Forms survey questions + conditional logic.  
- Create the initial survey script.  
- Map industries to target (Data, AI, Development, Cloud, Cybersecurity).  
- List target companies and professionals to contact.  
- Set timeline for surveys, interviews, and analysis windows.  
- Define your minimum viable dataset (e.g., 30 survey responses + 5 interviews).

**Deliverable:**  
A realistic project plan with dates, phases, and responsibilities.

---

---

## PHASE 4 — Testing & QA  
**Goal:** Ensure tools and survey logic work.  
**Mindset:** QA it like a data pipeline.

**Actions:**  
- Test Microsoft Form with 5–8 people.  
- Confirm conditional questions trigger correctly.  
- Test Excel export for readability.  
- Perform 1–2 test interviews to validate question clarity.  
- Improve wording, options, and logic.

**Deliverable:**  
Validated survey + interview script + clean test dataset.

---

## PHASE 5 — Data Collection  
**Goal:** Capture real-world insights from multiple sources.  
**Mindset:** The ingestion phase.

**Actions:**  
### Surveys  
- Distribute to companies, recruiters, professionals.  
- Track outreach and schedule follow-ups.  
- Document weekly response progress.

### Interviews  
- Conduct 5–10 interviews.  
- Summarize insights immediately after each call.  
- Save files as `interview_company_role_date.docx`.

### Public Data  
- Collect reports from Arbetsförmedlingen, SCB, industry organizations.

**Deliverable:**  
Complete raw dataset (surveys + interviews + public data).

---

## PHASE 6 — Data Processing & Analysis  
**Goal:** Convert raw data into insights.  
**Mindset:** Your ETL + analytics layer.

**Actions:**  
### Cleaning  
- Remove duplicates, empty entries.  
- Standardize industries.  
- Categorize skills (AI, Cloud, DevOps, Programming, Security).

### Quantitative Analysis  
- Count “increasing need” roles.  
- Identify most requested tech skills.  
- Chart decreasing roles.  
- Build bar charts, line charts, heatmaps.

### Qualitative Analysis  
- Analyze interview themes.  
- Highlight recurring signals.  
- Extract anonymized quotes.

**Deliverable:**  
Clean dataset + visual insights + trend summary.

---

## PHASE 7 — Reporting & Validation  
**Goal:** Transform findings into a decision-ready report.  
**Mindset:** Deliver strategic intelligence.

**Actions:**  
- Build final report (Executive Summary → Insights → Recommendations).  
- Add charts and visualizations.  
- Validate findings with 1–2 industry contacts.  
- Draft recommendations for new programs, expansions, phasing out roles, and skill updates.

**Deliverable:**  
Polished report + presentation slide deck.

---

## PHASE 8 — Project Closure  
**Goal:** Wrap up and prepare the project for repeatability.  
**Mindset:** Closure is part of the lifecycle.

**Actions:**  
- Evaluate what worked / what didn’t.  
- Document processes for future iterations.  
- Archive all files.  
- Capture lessons learned.  
- Confirm delivery to stakeholders.

**Deliverable:**  
Archived project folder + lessons-learned summary.
